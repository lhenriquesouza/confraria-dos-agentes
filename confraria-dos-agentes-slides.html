Contexto & Reasoning

📏 Janela de Contexto
DeepSeek V4 Pro: 1M tokens = livros inteiros, código de projeto completo. Contexto pequeno (8K-32K) = modelo "esquece" o começo da conversa.
⚙️ Reasoning
Low/Medium: resposta rápida, tasks simples. High/XHigh: tasks complexas (código, planejamento). Mais reasoning = mais caro e lento.

🧠 Módulo 2
🎁 Modelo Local com Ollama

Um modelo útil roda no próprio laptop — sem internet, sem custo, sem nuvem.

# Instalar
curl -fsSL https://ollama.com/install.sh | sh

# Rodar (modelo pequeno)
ollama run qwen2.5:1.5b
🔌 Offline
Funciona sem internet. Dados sensíveis não saem da máquina.
🇧🇷 Português
Responde em português. Pior que DeepSeek, mas grátis.
🧪 Prototipação
Ideal para testes, estudos, POCs sem gastar créditos.

💡 Conceito: Modelo local existe. Mas um modelo sozinho é só um chat no terminal — entra o harness.

🔧 Módulo 3
O Corpo: Harness

LLM sozinho gera texto. Harness faz ele AGIR.

❌ LLM Sozinho

  • Só gera texto
  • Responde perguntas
  • Segue prompts
  • Não lembra de nada
  • Ferramenta única
  • Só no navegador

✅ Com Harness

  • Executa comandos
  • Cria e edita arquivos
  • Pesquisa na web
  • Memória permanente
  • Skills (ensina uma vez)
  • Background, Telegram, cron

⚡ Analogia: LLM é o motor. Harness é o carro inteiro — volante, rodas, painel. Motor sozinho não te leva a lugar nenhum.

🎯 Módulo 6
/goal — Tarefa Complexa

1 agente, múltiplos passos em sequência. O agente gerencia a ordem sozinho.

📤 Prompt enviado

/goal "Faça uma pesquisa completa sobre o mercado de agentes de IA no Brasil em 2026.

  1. Pesquise as principais ferramentas, players, tendências e preços
  2. Crie um relatório executivo em markdown
  3. Converta para HTML estilizado (tema escuro, design profissional)
  4. Salve como relatorio-mercado-agentes.html
Pesquisa → Relatório MD → HTML final → Arquivo salvo

⚡ Quando usar /goal: tarefas lineares com dependência entre os passos. B precisa do resultado de A.

⚡ Módulo 6
/delegate_task — Implementação Paralela

N agentes independentes em paralelo, cada um com seu contexto isolado.

/grill-me → Spec → /to-tickets → /delegate_task
🤖 Agente 1
Coletor de preços: script Python que acessa API dos concorrentes e salva em JSON.
🤖 Agente 2
Dashboard HTML: página que carrega JSON e mostra gráficos de preços ao longo do tempo.
🤖 Agente 3
Alertas: compara preços e notifica via Telegram quando detecta mudança.

⚡ Quando usar /delegate_task: tarefas independentes. Todos rodam ao mesmo tempo. Limite: 3 simultâneas.

⚖️ Módulo 6
/goal vs /delegate_task

/goal/delegate_task
Agentes1Até 3
ParalelismoSequencialSim — cada um em isolamento
Quando usarTarefa única, passos dependentesTarefas independentes
Exemplo"Pesquise, analise e gere relatório""Pesquise 3 concorrentes ao mesmo tempo"
Custo1 agente filhoN agentes simultâneos

💡 Regra prática: Se B precisa do resultado de A → /goal. Se A, B, C são independentes → /delegate_task.

⏰ Módulo 8
Autonomia: Cron + Telegram

O agente trabalha sozinho. Você só recebe o resultado.

Briefing Matinal

hermes cron create "0 8 * * *" \
  -s briefing-matinal \
  -q "Briefing do setor de
    [MEU SETOR]. Foco: notícias,
    concorrentes, tendências."

# Disparo coletivo!
hermes cron run [JOB_ID]

☀️ 8h da manhã: briefing pronto no Telegram enquanto você toma café.

Autonomia no Dia a Dia

  • Cron + Telegram: briefing chega no celular
  • Cron + Google Workspace: arquiva emails, cria eventos
  • Múltiplos crons: briefing + monitoramento + alertas
# Gerenciamento
hermes cron list
hermes cron run ID
hermes cron pause ID
